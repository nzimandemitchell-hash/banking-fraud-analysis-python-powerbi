# banking-fraud-analysis-python-powerbi
An end-to-end data analytics project using Python and Power BI to analyze banking fraud transactions and deliver actionable business insights.


# Banking Fraud Detection Analysis

 Project Objectives

- Clean and prepare banking transaction data.
- Perform Exploratory Data Analysis (EDA) using Python.
- Identify fraud patterns and transaction trends.
- Analyze relationships between numerical variables using correlation analysis.
- Create an interactive Power BI dashboard for business users.
- Present actionable business insights and recommendations.

 Tools & Technologies

- Python
- Google Colab
- Pandas
- NumPy
- Matplotlib
- Seaborn
- Power BI
- DAX
- GitHub

##  Project Workflow

 1. Data Cleaning
- Imported the banking fraud dataset.
- Checked data types.
- Identified missing values.
- Removed unnecessary data issues.
- Created additional features such as Age Groups.

 2. Exploratory Data Analysis (EDA)
- Summary statistics
- Missing value analysis
- Fraud distribution
- Transaction amount analysis
- Risk score analysis
- Merchant category analysis
- Payment method analysis
- Transaction type analysis

3. Correlation Analysis
A correlation heatmap was created to identify relationships between numerical variables.

Key finding:
- Transaction Amount showed a moderate positive correlation with Risk Score (approximately **0.33**), indicating that larger transactions tend to receive higher risk scores.

4. Power BI Dashboard
An interactive dashboard was created to provide a high-level overview of banking fraud activity.

Dashboard features include:
- Total Transactions
- Total Transaction Amount
- Average Transaction Amount
- Fraud Rate
- Average Risk Score
- Fraud by Transaction Type
- Fraud by Merchant Category
- Interactive slicers

 Key Business Insights

- Most transactions are legitimate, with fraudulent transactions representing a smaller portion of the dataset.
- Higher transaction amounts generally have higher risk scores.
- Fraud trends vary across transaction types and merchant categories.
- Interactive filtering enables users to explore fraud patterns by merchant, transaction type, and province.
